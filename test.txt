1. リポジトリの作成と共有
1-1. GitHubで新しいリポジトリを作成
GitHubのウェブサイトにログインし、右上の 「+」 アイコンをクリックして 「New repository」 を選択。
リポジトリ名を入力し、Public または Private を選択します（Privateの場合、メンバーのみアクセス可能です）。
必要に応じて 「Add a README file」 にチェックを入れて作成を簡略化。
「Create repository」 ボタンをクリック。
1-2. メンバーの招待
作成したリポジトリのページに移動し、上部の 「Settings」 タブをクリック。
左サイドバーの 「Collaborators」 をクリック。
「Add people」 ボタンを押し、招待したいメンバーのGitHubユーザー名やメールアドレスを入力して招待。
2. GitHub Desktopでローカルにリポジトリをクローン
GitHub Desktopを開き、左上の 「File」 メニューから 「Clone repository」 を選択。
「URL」 タブを選択し、GitHubリポジトリのURLを貼り付けます。
保存先フォルダを選んで 「Clone」 ボタンをクリック。
これでリモートリポジトリがローカル環境にクローンされます。
3. コードの編集とコミット
ローカルリポジトリ内のコードを編集します（任意のコードエディタを使用）。
GitHub Desktopに戻ると、変更が表示されます。
変更を確認し、「Summary」 フィールドに変更内容を簡単に記述。
「Commit to main」 をクリックして変更をコミット。
4. リモートリポジトリへのプッシュ
コミット後、GitHub Desktopの右上にある 「Push origin」 ボタンをクリックすると、リモートリポジトリに変更が反映されます。
