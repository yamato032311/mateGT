1. リポジトリの作成と共有
1-1. GitHubで新しいリポジトリを作成
GitHubのウェブサイトにログインし、右上の 「+」 アイコンをクリックして 「New repository」 を選択。
リポジトリ名を入力し、Public または Private を選択します（Privateの場合、メンバーのみアクセス可能です）。
必要に応じて 「Add a README file」 にチェックを入れて作成を簡略化。
「Create repository」 ボタンをクリック。
1-2. メンバーの招待
作成したリポジトリのページに移動し、上部の 「Settings」 タブをクリック。
左サイドバーの 「Collaborators」 をクリック。
「Add people」 ボタンを押し、招待したいメンバーのGitHubユーザー名やメールアドレスを入力して招待。

4. リモートリポジトリへのプッシュ
コミット後、GitHub Desktopの右上にある 「Push origin」 ボタンをクリックすると、リモートリポジトリに変更が反映されます。

5. ブランチの作成とマージ
5-1. 新しいブランチの作成
左上の 「Current branch」 ドロップダウンから 「New Branch」 を選択。
新しいブランチ名を入力して 「Create branch」 をクリック。
各メンバーは、独自のブランチを作成して作業を進めると良いでしょう。

5-2. ブランチの変更をプッシュ
ブランチで作業が完了したら、通常のコミットと同様に変更をコミット。
右上の 「Push origin」 ボタンでブランチをリモートにプッシュ。

5-3. プルリクエストの作成
GitHubのウェブサイトにアクセスし、プッシュしたブランチに対して 「New pull request」 を作成。
変更内容を説明し、他のメンバーがレビューできるようにします。

5-4. プルリクエストのレビューとマージ
他のメンバーがプルリクエストを確認し、問題がなければ 「Merge pull request」 ボタンをクリックしてメインブランチに統合。

6. 最新の変更を取得する
チームメンバーが新しい変更をメインブランチにマージしたら、GitHub Desktopで 「Fetch origin」 をクリックし、最新の状態を取得します。
その後、「Pull origin」 ボタンを押してローカルリポジトリを最新に保ちます。
